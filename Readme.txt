Development, Brancing And Merging routines.

Development Branch
- Alle utviklere skal jobbe mot denne

Main Branch
- Ved verifisert test skal utvikleren merges til Main.

Release 1-n branch
- Deploymentansvarlige i forvaltning oppretter releaser fortløpende etterhvert som endringer er klare for release


Rutine (Alle utviklere skal følge):
1. Uvikle ny fix/løsning mot development.
2. Verifisere fix, og få godkjenning på denne.
3. Merge endringene inn i Main-branchen
4. Sende mail til support@communicate.no og oppgi følgende.

ReleaseNotes.txt
	- Prosjektnavn
		- Komponentnavn
	- Hva er endret siden forrige versjon.
		- Punktliste
		
Informer om følgende:
Ny release utviklet, verfisert og klar for utarbeidelse av ny release og deploy til øvrige kunder hvor komponenten finnes.


Deploymentansvarlige/Forvaltning.
	- Verifiserer ReleaseNotes.txt
	- Merger Main til nytt releasenr.
	- Deployer til øvrige kunder som har komponenten.
